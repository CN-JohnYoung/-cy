# 0. 卡牌模板
卡牌模板用于描述卡牌，卡牌实例在游戏开始前与游戏内可以随时创建与销毁，并且卡牌实例中的特效是可以添加删除与修改的

玩家角色同样是一种特殊的卡牌实例，其等级为玩家的酒馆等级，当前晶体矿，当前晶体矿上限，晶体矿最大上限，当前瓦斯与升本需要消耗的晶体矿作为特殊单位在单位数组中记录

每个玩家场上所有的卡牌普通单位之和有上限，记录在了全局常量里，卡牌入场时如果超过上限则会暂时溢出，但是卡牌无法获得普通单位

普通单位与特殊单位的函数通常分开写

# 1. 效果模板
特效模板用于描述特效，监听到对应event时便会尝试判断condition是否满足，如果满足则会selectTarget并尝试消耗cost来给每一个target执行operation。如果cost无法消耗，则中断效果。使用slot进行状态记录，不同的特效实例可以关联到同一个slot用于维护特效状态。

# 2. 商店阶段
通常来说，卡牌处于场上，卡池，商店，手牌，被合并等区域，现已维护了一个双向追踪机制，卡牌实例自己知道在哪个zone，也能通过玩家快速检索自己场上的卡牌，通常来说场上的卡牌内普通单位总和不能超过全局常量规定的值，但是给商店，卡池，手牌里添加单位通常不会有任何限制。当卡牌入场后总单位数量超过上限时，会溢出，但是后续场上的卡牌无法再获得普通单位。
